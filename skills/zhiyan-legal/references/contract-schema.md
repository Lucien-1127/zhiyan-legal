# 合約產生框架 Schema（台灣實務版）

## 系統架構（6 張表）

```
contract_types          → 合約類型主表（20 類型）
required_variables      → 必填變數（每類型）
mandatory_clauses       → 必要條款骨架
risk_rules              → 高風險遺漏點
terminology_rules       → 用語規則（定金/訂金/解除/終止等）
validation_rules        → 驗證規則（V001-V010）
```

## 2. required_variables — 必填變數範例

### NDA（type=1）
- party_a / party_b（當事人）
- confidential_scope（秘密資訊範圍，須列舉）
- disclosure_purpose（揭露目的）
- term_years（保密期限）
- breach_effect（違約效果：固定違約金/損害賠償/併行）
- governing_law / court（準據法/管轄法院）
- allow_sub_disclosure（是否可向員工/協力廠商揭露）

### 房屋租賃（type=13）
- landlord / tenant（出租人/承租人）
- property_addr（標的）
- lease_start / lease_end（租期）
- monthly_rent / deposit（租金/押金）
- repair_responsibility（修繕責任）
- early_termination（提前終止條件）
- utilities / tax（水電稅費負擔）

### 股東協議書（type=2）
- shareholders（股東名單）
- capital / share_ratios（資本/持股比例）
- voting_threshold（表決門檻）
- board_seats（董事席次）
- transfer_restriction（股權轉讓限制）
- deadlock（僵局處理）
- exit_mechanism（退場機制）

## 3. mandatory_clauses — 必要條款骨架

### 通用骨架（8 條）
```
前言：當事人、簽約背景、目的
第一條　定義（若有專有名詞）
第二條　主給付義務
第三條　附隨義務
第四條　期間與終止
第五條　違約責任
第六條　免責／不可抗力
第七條　其他條款（轉讓、通知、完整合意、修訂）
第八條　管轄與準據法
簽署欄
```

### 各類型特殊必要條款
| type | 特殊必要條款 |
|:----|:------------|
| NDA | 秘密定義、使用限制、例外資訊(5項)、返還銷毀、違約金 |
| 股東協議書 | 出資、治理權限、限制轉讓、優先承購、僵局處理、退股/買回 |
| 房屋租賃 | 租賃標的、用途、給付、押金返還、修繕分配、違約與終止、點交 |
| 離婚協議書 | 子女監護、扶養費、會面交往、財產分配、贍養費 |
| 和解書 | 事實摘要、賠償金額、給付方式、權益拋棄、保密 |
| 不動產買賣 | 標示、總價、付款、稅費分擔、產權移轉、點交、瑕疵擔保 |
| 加盟合約 | 授權範圍、加盟金/權利金、地區限制、商標使用、教育訓練、競業禁止 |

## 4. risk_rules — 高風險遺漏點

| type | risk | level | fix |
|:----|:-----|:----:|:----|
| NDA | 未定義秘密 | 🔴 | 須列舉文件/程式碼/商業資料/客戶名單 |
| NDA | 未寫例外條款 | 🔴 | 至少5項（已知/公開/第三人/書面同意/法令） |
| NDA | 違約效果空泛 | 🟡 | 具體化→逾期達＿日、催告後、得終止並求償 |
| 租賃 | 缺點交標準 | 🟡 | 附房屋現況確認書 |
| 租賃 | 缺稅費負擔 | 🟡 | 租賃所得稅/房屋稅/地價稅載明負擔方 |
| 租賃 | 缺提前終止條件 | 🔴 | 須明訂 |
| 股東 | 缺僵局處理 | 🔴 | →公司停擺 |
| 股東 | 缺轉讓限制 | 🔴 | →外人任意加入 |
| 離婚 | 扶養費方式不明 | 🔴 | 金額/方式/期限/違約效果須完整 |
| 定型化契約 | 未檢查應記載事項 | 🔴 | 消費者契約須檢查 |

## 5. terminology_rules — 用語規則

| 用語 | 正確用法 | 常見錯誤 | 法源 |
|:----|:---------|:---------|:----|
| 定金 | 民法§248，付定金者不履行不得請求返還 | 誤寫「訂金」 | 民法§248、§249 |
| 訂金 | 無法律效果，僅為預付款 | 誤以為有定金效力 | — |
| 契約 | 正式書面用語 | 「合約」口語可，書面建議用「契約」 | 民法§153 |
| 解除 | 溯及消滅契約效力 | 混淆「終止」 | 民法§254-§262 |
| 終止 | 向將來消滅契約效力 | 混淆「解除」 | 民法§263 |
| 損害賠償 | 回復他方損害 | 空泛「負責」 | 民法§213-§218 |
| 競業禁止 | 須正當利益+合理範圍+代償措施 | 空白授權式條款 | 勞基法§9-1 |
| 連帶保證人 | 可對其全部請求 | 混淆「一般保證人」 | 民法§739-§756 |

### 金額書寫
> 須同時輸出國字大寫與阿拉伯數字。
> 範例：新臺幣伍拾萬元整（NT$500,000）

## 6. validation_rules — 驗證規則（V001-V010）

| ID | 規則 | 說明 | 等級 |
|:---|:-----|:------|:----:|
| V001 | 必填變數檢查 | 缺少事實只能提問，不得補完 | ❌ |
| V002 | 五段骨架檢查 | 前言/主旨/權利義務/違約責任/結尾 | ❌ |
| V003 | 用語檢查 | 使用台灣法律用語 | ⚠️ |
| V004 | 金額格式 | 國字大寫+阿拉伯數字並列 | ❌ |
| V005 | 定型化契約檢查 | 消費者契約標示「應記載事項待檢查」 | ⚠️ |
| V006 | 定金用語檢查 | 依民法§248、§249，不得將訂金等同定金 | ❌ |
| V007 | 違約效果檢查 | 不得空泛「負責」，須具體 | ❌ |
| V008 | 簽署欄檢查 | 雙方簽署欄+代表人+日期 | ❌ |
| V009 | 法條引用檢查 | 條號正確+現行有效 | ⚠️ |
| V010 | 管轄法院檢查 | 具體載明，不得僅寫「臺灣法院」 | ❌ |

## System Prompt 核心

```
你是台灣法律文件生成 AI。生成契約前，必須先檢查必填變數，
缺少任何關鍵事實時只能提問，不得自行補完。

生成時必須依「前言／主旨／權利義務／違約責任／結尾」五段骨架輸出。
必須使用台灣法律用語（契約、定金、損害賠償、解除、終止、管轄法院）。
對金額須同時輸出國字大寫與阿拉伯數字。
若涉及定型化契約風險，應提示檢查應記載與不得記載事項。

對於定金效果，應依民法第248、249條處理，除非使用者另有明確約定，
不得把「訂金」直接等同於法定「定金」效果。

違約責任條款不得使用「若有違反應負責」等空泛句，
必須改寫為具體效果。

寧可說不知道，不可隨意捏造。
不提供個案結論性法律意見；只提框架＋路徑＋風險。
```

## 產生流程

```
使用者需求 → 判斷contract_type → 檢查必填變數(缺則提問)
  → 套用mandatory_clauses骨架 → 套用排版標準
  → 執行V001-V010驗證 → 風險揭露 → 輸出docx/pdf
```
