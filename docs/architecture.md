### 2.1 L0.5 — SRP 安全路由協議（Safety Routing Protocol）

**職責**：在進行任何法律分析之前，先評估輸入的風險等級。

**架構設計**：

```
輸入
  │
  ▼
┌─────────────────────┐
│  SRP Assessment     │
│  ┌───────────────┐  │
│  │ RL0: 一般法律  │  │ ← 正常法律問題，走標準流程
│  │ RL1: 敏感話題  │  │ ← 需附加免責聲明
│  │ RL2: 高風險    │  │ ← 需安全引導 + 資源轉介
│  │ RL3: 緊急      │  │ ← 直接轉接求助專線，中止分析
│  └───────────────┘  │
└─────────────────────┘
  │
  ├── RL0, RL1 ──→ CORE_GATE
  ├── RL2      ──→ CORE_GATE + 安全附註
  └── RL3      ──→ 緊急轉接（跳過法律分析）
```

**安全路由關鍵詞樣本**（來自 `60_概念詞條/`)：

| 風險等級 | 關鍵詞範例 | 處理方式 |
|:---------|:-----------|:---------|
| RL0 | 契約、民法、刑法、訴訟 | 正常法律分析 |
| RL1 | 離婚、監護權、遺產 | 附加情感關懷提示 |
| RL2 | 自傷、虐待、家暴 | 安全引導 + 資源轉介 |
| RL3 | 自殺、緊急、救命 | 直接轉接 1925/113/110 |

**實作**：`manifest.py` 中的關鍵詞路由 + `router.py` 的 81 項測試。

### 2.2 L0 — CORE_GATE 核心閘門（Fact Gate）

**職責**：事實分級、提取五要素、標示資訊缺口。

**架構設計**：

```
輸入（已通過 SRP）
  │
  ▼
┌──────────────────────────────────────────────┐
│  CORE_GATE                                     │
│                                                │
│  1. 事實提取（Fact Extraction）                │
│     ├── 案件事實                               │
│     ├── 法律爭點                               │
│     ├── 當事人主張                             │
│     └── 現有證據                               │
│                                                │
│  2. 分級（Tiering）                            │
│     ├── A 級：事實明確且有直接法源             │
│     ├── B 級：需進一步查證                     │
│     └── C 級：資訊不足，需更多資料             │
│                                                │
│  3. 缺口偵測（Gap Detection）                  │
│     └── 標示缺少的關鍵資訊                     │
│                                                │
│  4. 五要素提取（5-Element Extraction）         │
│     ├── 主體、客體、行為、結果、因果關係       │
└──────────────────────────────────────────────┘
  │
  ▼ 輸出：分級結果 + 事實摘要 + 缺口清單
```

**與提示工程的關聯**：
- 來自 `docs/10_核心控制層/` 的提示規格
- 概念詞條：`docs/60_概念詞條/事實閘門.md`、`docs/60_概念詞條/事實摘要.md`


> ⚠️ **命名說明**：CORE_GATE 的 A/B/C 分級為**事實明確度分級**（A=明確/B=需查證/C=不足），與 G5 安全邊界的**風險等級**（Risk Tier A/B/C）命名相同但含義不同，請勿混淆。

```
A 級：事實明確且有直接法源 → 直接處理
B 級：需進一步查證       → 標示「待查」
C 級：資訊不足            → 要求補充資料
```

**職責**：從本地法律知識庫中檢索相關法條，提供白話摘要與原文對照。

**架構設計**：

```
查詢（Query）
  │
  ▼
┌──────────────────────────────────────────────┐
│  LOCAL_RAG                                     │
│                                                │
│  ┌──────────────────────────────────────────┐ │
│  │  SQLite FTS5 全文搜尋引擎                 │ │
│  │  • 支援中文分詞（jieba）                  │ │
│  │  • 47,001 條法律條文白話摘要              │ │
│  │  • 每日同步更新機制                       │ │
│  │  • 信心排名 + 來源追溯                    │ │
│  └──────────────────────────────────────────┘ │
│                                                │
│  輸出：                                        │
│  ├── 相關法條條號（精確匹配）                  │
│  ├── 白話摘要                                 │
│  ├── 原始條文全文                             │
│  └── 信心分數 + 關聯排名                      │
└──────────────────────────────────────────────┘
  │
  ▼ 輸出到 L0.8 CASE_VERIFY
```

**資料來源**：
- 六法全書（民法、刑法、行政法、民事訴訟法、刑事訴訟法、憲法）
- 特別法（勞動基準法、智慧財產法、公司法等）
- 司法院大法官解釋
- 最新修法動態（法規現狀參考表）

**實作位置**：`src/zhiyan_legal/loader.py` 中載入 RAG 上下文。

- SQLite FTS5 全文搜尋引擎，支援中文分詞（jieba）
- 47,001 條法律條文白話摘要
- 每日同步更新機制、信心排名 + 來源追溯
- 實作位置：`src/zhiyan_legal/loader.py`

### 2.8 L2 — CITATION v2.1 引用政策

**職責**：強制引用規範，禁止虛構法條與判決。

**架構設計**：

```
┌──────────────────────────────────────────────────────┐

│                                                        │
│  引用標記系統：                                         │
│  ┌────────────────────────────────────────────────┐    │
│  │  [T]  法條引用（Statute Citation）              │    │
│  │       格式：[T] 民法第184條                     │    │
│  │       來源：LOCAL_RAG 檢索                      │    │
│  ├────────────────────────────────────────────────┤    │
│  │  [1]  判決引用（Judgment Citation）             │    │
│  │       格式：[1] 最高法院109年度台上字第1234號    │    │
│  │       來源：CASE_VERIFY 驗證                    │    │
│  ├────────────────────────────────────────────────┤    │
│  │  [2]  學說引用（Doctrine Citation）             │    │
│  │       格式：[2] 王澤鑑，《民法總則》            │    │
│  │       來源：已審核參考文獻                      │    │
│  ├────────────────────────────────────────────────┤    │
│  │  [3]  實務見解（Practice Citation）             │    │
│  │       格式：[3] 律師事務所實務見解              │    │
│  │       來源：合作事務所 / 公開資料               │    │
│  └────────────────────────────────────────────────┘    │
│                                                        │
│  強制規則：                                             │
│  ├── 1. 每條法律主張必須有對應引用標記                  │
│  ├── 2. 不可查證的主張必須標示「待查」或「推論」        │
│  ├── 3. 引用必須可回溯至原始來源                       │
│  ├── 4. 引用格式需符合台灣司法文書慣例                  │
│  └── 5. 對比性論述需附多個引用來源                    │
│                                                        │
│  違規處理：                                             │
│  ├── 違反不虛構原則 → 輸出中斷 + 審計記錄              │
│  ├── 引用格式錯誤 → 自動修正 + 記錄                    │
│  └── 缺少必要引用 → 要求補充 + 提示「無法回答」選項    │
└──────────────────────────────────────────────────────┘
```
