# 98_更新說明_v3.07 — MCP 基礎設施 + 憲判檢查層

本檔記錄 2026-06-27 智研 v3.06 → v3.07 的架構升級：MCP 協議全面導入、憲法法庭判決強制檢查層上線、舊指令集整合準備就緒。

## 1. 一句話結論

**MCP 取代了所有需憑證的官方 API**，同時新增憲判強制檢查層修正 L0.8 的 QC 盲點；舊版 Drive 指令集（v3.00.3）已完成差異分析，待整合項目已列管。

## 2. 核心變更

### 2.1 MCP 基礎設施（全新）

| # | 項目 | 說明 | 狀態 |
|:-:|:--|:--|:--:|
| 1 | MCP Taiwan Legal DB | 取代 `judicial_api.py` — 免憑證、24/7、8 工具即時檢索 | ✅ 22/24 壓力測試通過 |
| 2 | GCP BigQuery MCP | 資料倉儲查詢 | ✅ |
| 3 | GCP Cloud Storage MCP | 物件儲存操作 | ✅ |
| 4 | GCP Cloud Run MCP | Serverless 部署管理 | ✅ |
| 5 | GCP Cloud Logging MCP | 日誌查詢與監控 | ✅ |
| 6 | GCP Compute Engine MCP | VM 執行個體管理 | ✅ |
| 7 | GCP Resource Manager MCP | 專案與 IAM 管理 | ✅ |
| 8 | Token 自動刷新機制 | GCE metadata 每 30 分鐘自動刷新 GCP OAuth token | ✅ cron `*/30 * * * *` |
| 9 | `gcp-mcp-setup` 技能 | 完整設定步驟、驗證程序、故障排除 | ✅ 新建 |

### 2.2 憲法法庭判決強制檢查層（L0.8 強化）

| # | 項目 | 說明 |
|:-:|:--|:--|
| 1 | 強制檢查觸發條件 | 7 組：基本權條文出現、釋字／憲判字號被引用、系爭法律涉及合憲性、比例原則論述、判決違憲效力陳述、實務見解引用、刑事法條適用 |
| 2 | 執行方式 | L0.8 首個步驟即查 `mcp-taiwan-legal-db search_judgments`，以「憲法法庭」「憲判」為關鍵字；查到的憲判字號必須寫入輸出中 |
| 3 | 對應舊指令 | 源自 v3.00.3 `04_人格_教學_TUTOR.txt` 的「結構化分析架構」中「憲法層次—審查密度選擇」指令 |
