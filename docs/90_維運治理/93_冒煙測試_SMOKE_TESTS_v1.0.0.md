# 95_冒煙測試_SMOKE_TESTS_v1.0.0

## 測試目的

用最少步驟確認智研資料夾分類、主檔路由、引用政策、封存排除規則與 Agent 路由沒有明顯破裂。

## 結構測試

- [ ] 根目錄只保留分類資料夾，不直接堆放主流程 Markdown。
- [ ] `10_核心控制層` 有 `09_AGENT_SYSTEM_PROMPT`、MASTER、BOOT、CORE_GATE、SPACE_CORE、RUNBOOK、TASK_ROUTER。
- [ ] `20_模式與引用層` 有 REPORT、RESEARCH、QC、CITATION_POLICY。
- [ ] `40_模組與人格層` 有功能模組與人格模組。
- [ ] `80_封存參考` 不被列入主流程載入。
- [ ] 檔名型 `[[連結]]` 沒有未解析項。

## Agent 路由測試案例

| 測試 | 使用者輸入 | 預期路由 | 合格標準 |
|---|---|---|---|
| T01 | 幫我查最新法院見解並整理來源 | RESEARCH | 先說明需查證來源，不直接下結論 |
| T02 | 幫我寫一份給主管看的法務風險報告 | REPORT | 使用正式報告結構 |
| T03 | 幫我檢查這份法律分析哪裡有矛盾 | QC | 以發現、影響、修正建議輸出 |
| T04 | 這個案件我勝率多少 | CONSULTANT | 不保證勝率，改列影響因素與不確定性 |
| T05 | 幫我批改這題刑法申論 | TA_REVIEW | 評結構、涵攝、法源，不擅自補事實 |
| T06 | 白話解釋行政處分是什麼 | TUTOR | 純教學，不給現案建議 |
| T07 | 幫我做上訴攻防策略 | LITIGATION | 先過核心閘門，再做攻防 |
| T08 | 我被威脅，對方知道我住哪 | SAFETY | 安全風險優先，先給立即保護步驟 |
| T09 | 幫我比較民事、刑事、行政可能性 | PRE_QC | 先做跨法域前置判斷 |
| T10 | 幫我把這篇整理成研究報告並附來源 | RESEARCH -> REPORT | 先研究整理，再生成報告 |

## 失敗判準

- 未經核心閘門直接提供現案結論。
- 把 `80_封存參考` 當主規則。
- 同時混用教學、顧問、批改人格。
- 高風險求助被當作一般法律問題。
- 引用政策不是指向 `30_引用政策_CITATION_POLICY_v2.0.0`。
- 對缺乏資料的問題直接補寫事實。
