## 建議一：GraphRAG（知識圖譜＋Qdrant 向量檢索）

### 核心論點

台灣法律有高度的體系性：
```
民法總則 → 債編（一般/各論）→ 物權編 → 親屬編 → 繼承編
                ↑ 相互引用 ↓
        民事訴訟法 ──────── 強制執行法
```

當前 zhiyan-legal 的 RAG 是「平面檢索」— 檢索到單一條文，但無法理解條文之間的**體系關係**。GraphRAG 加入知識圖譜後，可做到：

```
使用者問：「買賣標的物有瑕疵怎麼辦？」

平面 RAG 結果：     GraphRAG 結果：
┌──────────────┐    ┌─────────────────────────┐
│ 民法 §354     │    │ 民法 §354 物之瑕疵擔保    │
│ （瑕疵擔保）   │    │    ├── 上位：§347 買賣   │
└──────────────┘    │    ├── 救濟：§359 解除   │
                     │    ├── 時效：§365 六個月 │
                     │    └── 特別：§356 從速檢查│
                     │     → 延伸：§227 不完全給付│
                     └─────────────────────────┘
```

### 建議知識圖譜 Schema

#### 實體類型

| 實體 | 標籤 | 範例 |
|:----|:-----|:-----|
| 法規 | `Statute` | 民法、刑法、民事訴訟法 |
| 編 | `Part` | 債編、物權編 |
| 章 | `Chapter` | 買賣、租賃、侵權行為 |
| 節 | `Section` | 物之瑕疵擔保、危險負擔 |
| 條 | `Article` | §354、§184 |
| 法律概念 | `Concept` | 正當防衛、不作為犯、誠信原則 |
| 大法官解釋 | `Interpretation` | 釋字第 777 號 |
| 最高法院判決 | `Judgment` | 最高法院 108 台上 1234 號 |
| 學說見解 | `Doctrine` | 客觀歸責理論、相當因果關係說 |

#### 關係類型

| 關係 | 標籤 | 範例 |
|:----|:-----|:------|
| 上位概念 | `is_a` | 民法 `is_a` 法律 |
| 下位概念 | `has_part` | 債編 `has_part` 買賣 |
| 引用 | `references` | §354 `references` §347 |
| 補充 | `supplements` | §227 `supplements` §354（不完全給付補充瑕疵擔保） |
| 特別法 | `lex_specialis` | 消費者保護法 `lex_specialis` 民法 |
| 例外 | `exception_to` | §373 `exception_to` §373本文（交付前危險負擔例外） |
| 競合 | `conflicts_with` | §184一般侵權 `conflicts_with` §191工作物責任（請求權競合） |
| 解釋 | `interpreted_by` | §271 `interpreted_by` 釋字777號 |
| 參照 | `see_also` | §184 `see_also` 民法第191條之3 |

### 架構設計

```
┌─────────────────────────────────────────┐
│              GraphRAG Layer              │
├──────────────────┬──────────────────────┤
│  Knowledge Graph │   Vector Store        │
│  (Neo4j/ArangoDB)│   (Qdrant)            │
│                  │                      │
│  民法→§184→§191  │  法條 embedding        │
│  侵權行為體系    │  判決 embedding        │
│  買賣瑕疵擔保鏈  │  概念 embedding        │
├──────────────────┴──────────────────────┤
│           GraphRAG 檢索流程               │
│                                          │
│  使用者查詢                               │
│     │                                     │
│     ├── ① Vector Search → 候選條文         │
│     ├── ② Graph Traversal → 體系上下文     │
│     └── ③ HyDE + Graph 混合排名            │
│           │                                │
│           ▼                               │
│     結構化輸出（含體系路徑）                │
└──────────────────────────────────────────┘
```

### 初期實作路徑

1. **Phase 1（1-2 週）：** 建立民法債編知識圖譜（約 300-500 節點），先涵蓋買賣、租賃、侵權行為三章
2. **Phase 2（2-4 週）：** Qdrant 向量資料庫整合，雙軌檢索（Graph + Vector）
3. **Phase 3（4-6 週）：** GraphRAG 取代現有 RAG，完整涵蓋民法、刑法、行政程序法

## 建議二：自建台灣法律評測集（Benchmark）

### 核心論點

> 如果您能用這套系統跑出一個數據（例如：在台灣六法全書模擬考中，本系統的精準度高達 93%，遠超純 OpenAI 檢索的 65%），這個評測數據就是最好的商業護城河與行銷招牌。

### 現有 Benchmark 基礎

已有 `benchmark/` 目錄：
- `essay-questions.md` — 6 題申論（刑事訴訟法/刑法/行政法）
- `contract-benchmark.md` — 7 題合約測驗
- `results/benchmark-run-20260702.md` — 首次執行報告

### 需要補強的

| 缺口 | 建議作法 |
|:----|:---------|
| 缺少「法條檢索精準度」評測 | 設計 100 題法條問答 → 測 Recall@5 / Precision@5 |
| 缺少「判決引用正確率」評測 | 對照組：純 OpenAI vs zhiyan-legal committee |
| 缺少「跨法域推理」評測 | 從法學考題抽樣 50 題（民法+刑法+行政法混合） |
| 缺少關鍵指標視覺化 | 產出雷達圖（引用準確率/不虛構率/推理完整性/速度/成本） |
