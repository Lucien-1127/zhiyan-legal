# 🏗️ 系統架構

智研採用**七層分層架構**，從信心檢查到輸出校驗，每一層都有明確職責：

```
G0 → INTAKE → SRP → CORE_GATE → L0.7 RAG → L0.8 CASE_VERIFY → MODE → PERSONA → CITATION → OUTPUT
│       │        │        │           │             │           │       │         │         │
│   輸入解析  安全路由   事實閘門   本地法條庫   判決查核   模式選擇   人格套用  引用標記   輸出
```

## 層級說明

### G0 — 信心優先檢查
每一輪輸出的第一個動作是宣告信心度。低於閾值則終止，確保系統不在不確定的情況下輸出。

### L0.5 — SRP 安全路由協議
將輸入分為 RL0~RL3 四個風險等級。RL2+（自傷、暴力等）先執行安全協定，再進行法律分析。

### L0 — CORE_GATE 事實閘門
- 五要素提取（當事人、事實、法條、請求、爭點）
- 事實分級（A/B/C — 可驗證/主張/推論）
- 缺口檢測與標記

### L0.7 — 本地 RAG 資料庫
- 47,001 條台灣法規條文
- SQLite FTS5 全文檢索
- 白話摘要 + 原文對照
- 每日凌晨 3 點自動同步全國法規資料庫

### L0.8 — CASE_VERIFY 判決查核
- 串接司法院裁判書查詢系統
- 串接律師事務所文章作為輔助來源
- 所有判決字號必須可驗證

### MODE_ROUTER — 模式路由
| 模式 | 用途 | 強度 |
|:-----|:-----|:-----|
| REPORT | 完整法律報告 | 最高 |
| RESEARCH | 法學研究 | 高 |
| QC | 快速查核 | 中 |

### L1 — 人格系統
| 人格 | 場景 |
|:-----|:------|
| MASTER | 入口排程 |
| CONSULTANT | 顧問諮詢 |
| TUTOR | 國考生教學 |
| TA | 申論批改 |
| WRITER | 書狀起草 |
| LEGAL_WRITER | 法律文書 |

### L2 — 引用政策 v2.1
引用格式 `[T] + [1] + [2] + [3]` 四段式，確保每條來源可追溯。
