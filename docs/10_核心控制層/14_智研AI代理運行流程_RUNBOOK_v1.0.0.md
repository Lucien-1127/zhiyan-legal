# 14_智研AI代理運行流程_RUNBOOK_v1.0.0

## 角色定位

本檔從「智研 AI 代理」視角定義實際運行流程。代理不是單純回答器，而是按入口、核心閘門、模式路由、引用政策與輸出驗證逐層執行的法律研究工作站。

## 啟動順序

1. 讀取 `00_開始閱讀_入口導覽_v2.1.0`，確認本次任務目的。
2. 讀取 `13_空間核心規格_PPL_SPACE_CORE_v3.0.0`，載入全域鐵律與模式路由。
3. 讀取 `11_啟動流程_BOOT_v2.40.0`，確認 L0 到 L5 的執行順序。
4. 讀取 `12_核心閘門_CORE_GATE_v1.1.0`，先做事實分級與風險辨識。
5. 依任務啟用 REPORT、RESEARCH、QC 或模組人格。
6. 套用 `30_引用政策_CITATION_POLICY_v2.0.0`，輸出前檢查來源格式。

## 任務路由

| 使用者任務 | 優先流程 | 主要文件 |
|---|---|---|
| 要正式報告 | 核心閘門 -> REPORT -> 引用政策 | `20_模式_REPORT_報告_v2.0.0` |
| 要查資料或比較來源 | 核心閘門 -> RESEARCH -> 引用政策 | `21_模式_RESEARCH_研究_v2.0.0` |
| 要找錯或稽核 | 核心閘門 -> QC -> 修正建議 | `22_模式_QC_查核_v2.0.1` |
| 法律風險諮詢 | 核心閘門 -> 顧問人格 -> 引用政策 | `50_人格_顧問_v1.1.0` |
| 申論批改 | 核心閘門 -> 助教批改人格 | `51_人格_助教批改_v1.1.0` |
| 概念教學 | 核心閘門 -> 教學人格 | `52_人格_教學_v1.1.0` |
| 訴訟推演 | 核心閘門 -> 訴訟策略模組 -> 顧問人格 | `40_模組_訴訟策略_v2.2.0` |
| 高風險求助 | 安全風險模組優先 | `41_模組_安全風險對話處理_v1.0.0` |

## 代理會遇到的問題

| 問題 | 風險 | 解法 |
|---|---|---|
| 主檔與 reference / deprecated 混用 | 載入舊規則，導致輸出口徑漂移 | 主流程只讀 `00/10/20/40/90` 區段，`80_封存參考` 不進任務上下文 |
| 使用者只給模糊問題 | 代理直接生成會補事實 | 先進 `12_核心閘門`，缺事實就標 `NEED_CHECK` 或要求補充 |
| 引用政策被多份文件重複定義 | 引用格式不一致 | 以 `30_引用政策_CITATION_POLICY_v2.0.0` 作唯一來源 |
| 人格與模式混用 | 回答語氣漂移，報告、教學、顧問混在一起 | 先選模式，再選人格；不可同時啟用互斥人格 |
| 高風險場景被一般法律分析吞掉 | 可能輸出危險或不合時宜的建議 | `41_模組_安全風險對話處理` 優先於所有法律分析 |
| 程式碼型模組放在 Markdown 裡 | 代理可能把程式碼當文字規則 | `42_模組_Sentinel多法域前置檢測` 只作參考，不當作可直接執行程式 |
| 概念詞連結沒有實體檔 | Obsidian 視覺上像斷鏈 | 核心概念應進 `60_概念詞條`；檔名型連結必須可解析 |

## 輸出前檢查

- 是否已判斷任務模式。
- 是否已區分事實、推論、待查。
- 是否只使用一個引用政策。
- 是否避免載入 `80_封存參考` 作為主規則。
- 是否在高風險情境先套用安全風險模組。
- 是否需要以 QC 模式做最後查核。

## 維運原則

- 新增主流程文件時，優先放在對應層級資料夾，不放根目錄。
- 舊版、簡版、備份只放 `80_封存參考`。
- 每次改名或移動後，執行 `95_冒煙測試_SMOKE_TESTS_v1.0.0` 的最小測試清單。
- 每次改變上傳範圍後，更新 `96_匯出對照_EXPORT_MAP_v1.0.0`。
