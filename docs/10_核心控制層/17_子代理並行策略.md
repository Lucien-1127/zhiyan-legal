# 子代理並行策略 v1.0

## 為什麼需要子代理

目前 zhiyan-legal 的七層架構是**全同步順序執行**：

```
L0.5 → L0 → L0.7 RAG → L0.8 TYPE-S → MODE_ROUTER → 功能執行 → 輸出
                                                          │
              引證驗證 law.moj.gov.tw ──────────────────────┤
              判決查詢 judgment.judicial.gov.tw ────────────┤（全部序列）
              實務文章交叉比對 ─────────────────────────────┤
```

隨著功能擴充，每個環節的等待時間累加，主要瓶頸在**聯網查詢**和**多角色模擬**。

## 五個並行機會

### 1. 法庭模擬三方並行 🔥（最有感）

**目前**：法官 → 檢察官 → 辯護人，一個跑完換下一個，3 次序列 LLM 呼叫
**優化**：三個角色各由一個子代理獨立準備，最後再合成庭審記錄

```
┌─ sub_agent_judge.py ───── 準備法官視角的案卷摘要
├─ sub_agent_prosecutor.py ─ 準備起訴論告
├─ sub_agent_defender.py ─── 準備辯護要旨
└─ main_agent ───────────── 合成三方攻防 → 輸出庭審記錄

加速比：3× 序列 → 1× 平行
```

### 2. 引證驗證搜尋並行 🟡（加速最明顯）

**目前**：law.moj.gov.tw → judgment.judicial.gov.tw → 律師文章，逐個查
**優化**：三個來源同時查，結果回來後一次比對

```
┌─ sub_agent_law.py ──────── 查 law.moj.gov.tw → 條文原文
├─ sub_agent_judgment.py ─── 查 judgment.judicial.gov.tw → 判決見解
├─ sub_agent_practice.py ─── 查律師事務所文章 → 實務現狀
└─ main_agent ───────────── 三路結果交叉比對 → 發現分歧點

加速比：3× 序列 → 1× 平行
```

### 3. TYPE-S QA 分離 🟡

**目前**：同一個 agent 自我審查（可能 blind spot）
**優化**：由獨立的 QA 子代理審查主 agent 的輸出

```
main_agent 產出草稿
    │
    └─→ sub_agent_qa.py（獨立 context）
         → 只做法條正確性檢查
         → 只做引用可追溯性檢查
         → 只做格式一致性檢查
         → 回傳驗證報告
    │
main_agent 根據 QA 報告修正 → 最終輸出

blind spot 防禦：QA agent 不知道主 agent 的思考過程
```

### 4. 多法域平行研究 🟢

**目前**：同一個 agent 逐一處理刑法、民法、行政法
**優化**：按法域拆給專門子代理

```
┌─ sub_agent_criminal.py ─── 刑法爭點分析
├─ sub_agent_civil.py ────── 民法爭點分析
├─ sub_agent_admin.py ────── 行政法爭點分析
└─ main_agent ───────────── 整合跨法域 → 發現法域衝突點
```

### 5. RAG + 聯網平行抓取 🟢

**目前**：先查本地 RAG，沒有的話再聯網
**優化**：同時查，誰先回來就用誰，加速查詢

```
┌─ sub_agent_rag.py ──────── 查本地 SQLite RAG（毫秒級）
├─ sub_agent_online.py ───── 查 law.moj.gov.tw（網路延遲）
└─ main_agent ───────────── 比對兩者，不一致時標示分歧
```

## 實作順序（依效益排列）

| 優先 | 項目 | 效益 | 實作難度 |
|:----:|------|------|---------|
| 🥇 | 法庭模擬三方並行 | 庭審速度 ×3 | 中 |
| 🥇 | 引證驗證搜尋並行 | 查詢速度 ×3 | 低 |
| 🥈 | TYPE-S QA 分離 | 品質提升 | 中 |
| 🥉 | 多法域平行研究 | 完整度提升 | 高 |
| 🥉 | RAG + 聯網平行 | 速度 ×2 | 低 |

## Hermes 子代理技術

zhiyan-legal 跑在 Hermes Agent 上，使用 `delegate_task`：

```python
from hermes_tools import delegate_task

# 平行查三個來源
results = delegate_task(tasks=[
    {"goal": "查全國法規資料庫 XXX 條", "toolsets": ["web"]},
    {"goal": "查司法院判決書 XXX 號", "toolsets": ["web"]},
    {"goal": "查律師文章關於 XXX 實務見解", "toolsets": ["web"]},
])

# 每個 result 是子代理的摘要
# 主代理再綜合成完整分析
```

注意事項：
- 子代理不知道主 session 的對話，需要傳遞完整的 task context
- 每個子代理獨立 terminal session，不會互相干擾
- 子代理回傳的是 summary，不佔用主 context window 的中間步驟
