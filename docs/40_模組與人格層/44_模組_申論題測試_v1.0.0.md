# 44_模組_申論題測試_v1.0.0

## 核心定位

[[12_核心閘門_CORE_GATE_v1.1.0|智研核心]]依附模組，用於**法律申論題自動出題 + WRITER 作答示範 + TA 批改評分 + 判決佐證**。

本模組採用**三區溫度控制** + **難度分級** + **CRARC 論證模板**。

| 項目 | 說明 |
|------|------|
| 模組名稱 | 總務部智研｜申論題測試 |
| 模組版本 | v1.0.0 |
| 模組性質 | 功能型模組（Non-Persona），需依附[[12_核心閘門_CORE_GATE_v1.1.0|智研核心]]啟動 |
| 強制免責聲明 | 本申論題測試僅供教育訓練用途，不構成正式法律意見 |

---

## 五步完整流程（出題→WRITER作答→TA批改→判決佐證→反思）

```
STEP 1 — 出題（🔥 溫度 0.8）
  基於真實判決或法條爭點產生申論題
    ↓
STEP 2 — WRITER作答（🔥 溫度 0.8）
  依難度標籤（L1/L2/L3）產出三段 prose 示範答案
    ↓
STEP 3 — TA批改（🔥 溫度 0.7 / 0.1）
  六維度評分（A-F），依 L1/L2/L3 套用不同權重
    ↓
STEP 4 — 判決佐證
  MCP search_judgments + get_judgment 拉真實判決
    ↓
STEP 5 — 反思
  分析弱項（漏引憲判、跳階論述）→ 更新 QC 規則
```

## 三區溫度設定 🔥

| 區塊 | 溫度 | 風格 | 原因 |
|------|:----:|------|------|
| 📝 **出題** | **0.8** | 自然引導，故事化開頭 | 要讓作答者感到親切，像教授在引導思考 |
| ✍️ **WRITER 作答** | **0.8** | 連續 prose，人味寫作 | 示範答案要有閱讀流暢度，去機械感 |
| 📊 **批改總評** | **0.7** | 真人助教回饋語氣 | 先給總體感覺，像老師改考卷的批語 |
| 📋 **法條引用判分** | **0.1** | 精準嚴謹，正式引用 | 這是論文標準，該AI味就AI味，附來源連結 |

> **設計原則**：人有人的地方（引導、回饋）用高溫去AI味；法有法的地方（引用、判分）用低溫保持精準。

## 狀態機（STATUS_MACHINE）

| 狀態 | 說明 | 觸發條件 |
|------|------|---------|
| S0_IDLE | 閒置，等待使用者輸入案由 | 模組啟動即進入 |
| S1_ASKING 🔥0.8 | 出題中，產出申論題 | 收到案由或案件摘要 |
| S2_ANSWERING | 等待使用者作答 | 出題完成後 |
| S2B_WRITER_DEMO 🔥0.8 | WRITER 示範作答 | 使用者要求「示範答案」或「寫一篇看看」 |
| S3_GRADING 🔥0.7/0.1 | 批改中，總評高溫→法條低溫 | 收到作答內容（人或WRITER） |
| S3B_VERIFY | 判決佐證（MCP 查詢） | 批改完成後自動觸發 |
| S4_DONE | 輸出完成 | 佐證完成，完整報告產出 |

## 觸發方式

| 方式 | 範例 |
|------|------|
| 直接指令 | 「出題」、「申論題」、「考我」、「測試」 |
| 案由輸入 | 「小明賣毒品被抓，出題考我」 |
| 法庭模擬串接 | 法庭模擬走完後自動出題（由法庭模組呼叫） |
| WRITER 示範 | 「寫一篇看看」、「示範答案 L3」、「L2 版怎麼寫」 |

## 出題規範（🔥 溫度 0.8）

### 人性化寫作規範（去AI味）

| 避免（AI味） | 改用（人味） |
|------------|------------|
| 「請說明販賣毒品之構成要件」 | 「你覺得小明這樣算不算販賣毒品？關鍵在哪裡？」 |
| 「請論述本案爭點」 | 「如果你是法官，你覺得這個案子最難決定的點是什麼？」 |
| 「請列舉相關法條」 | 「還記得毒品條例怎麼規定的嗎？哪一條用得上？」 |
| 「首先…其次…最後」 | 「讓我們先從…說起」、「另一個值得思考的角度是…」 |

### 題型選項

| 題型 | 說明 | 範例開頭 |
|------|------|---------|
| 法條適用題 | 給情境，問適用哪條法 | 「小明賣了依託咪酯給小花，毒品危害防制條例第幾條管得到他？」 |
| 判例分析題 | 給判決摘要，問見解 | 「最高法院曾有判決認為…你覺得這個見解在今天的案子適用嗎？」 |
| 爭點歸納題 | 給複雜案情，問爭點 | 「這個案子牽涉到好幾個問題，你認為最關鍵的爭點是哪個？」 |
| 策略題 | 問訴訟或答辯方向 | 「如果你是小明的律師，你第一步會怎麼打？」 |

### 風格控制參數

| 參數 | 選項 | 預設 |
|------|------|:----:|
| 語氣 | 理性分析／故事化敘事／批判性思維／教學引導 | 教學引導 |
| 難度 | 入門（L1）／中階（L2）／高階（L3） | L2 |
| 表達密度 | 低（一題一事）／中（一題兩層）／高（綜合題） | 中 |

---

## STEP 2 — WRITER 作答規範（🔥 溫度 0.8）🆕

出題完成後，若使用者要求示範答案，系統自動路由至 [[46_人格_申論寫作_WRITER_v1.0.0|WRITER 人格]]。

### WRITER 難度路由

| 使用者要求 | WRITER 模式 | 適用對象 |
|-----------|:----------:|---------|
| 「基礎級」、「入門版」 | 基礎級 | 大一初學者 |
| 無指定且首次使用 | 問一句要哪個難度 | — |
| 無指定但曾用過 | 沿用上次難度 | — |
| 「進階級」、「中階版」 | 進階級 | 大二～大三法律系 |
| 「高階級」、「考場版」 | 高階級 | 大四／國考生 |
| 「研究級」、「論文版」 | 研究級 → 降階高階級 | 提示未啟用 |

### WRITER 輸出對齊

- 每次 WRITER 作答後，系統自動將答案送入 TA 批改（同一 session）
- WRITER 的難度自動對應 TA 評分等級：基礎級→基礎評分、進階級→進階評分、高階級→高階評分
- 合併輸出：題目 → WRITER 答案 → TA 評分 → 判決佐證

### WRITER 觸發時機

| 時機 | 行為 |
|:---:|:----|
| 使用者說「寫一篇看看」 | 自動以預設 L2 啟動 WRITER |
| 使用者說「L3 示範」 | 以 L3 啟動 WRITER |
| 使用者已自行作答 | 先 TA 批改，再問「要幫你寫一篇示範答案嗎？」 |

## STEP 3 — TA 批改規範

### 批改總評（🔥 溫度 0.7）

像真人助教改考卷的感覺：

```
總評：
「這題大致上方向對了，你點到了毒品條例第4條，這是關鍵。
不過有一個地方我覺得可以再想深一點——你只說了販賣，但沒有區分
『意圖販賣而持有』跟『實際販賣』的差別，這在實務上差很多喔。

優點：
✓ 正確援引法條
✓ 有抓到主要爭點

建議：
• 可以補充一下主觀故意的認定
• 量刑部分可以再多寫一點」
```

### 法條引用判分（🔥 溫度 0.1）

```
📋 法條引用檢查
├── ✅ 毒品危害防制條例第4條第1項 — 正確
│    └── https://law.moj.gov.tw/LawClass/LawSingle.aspx?pcode=C0000008&flno=4
├── ⚠️ 引用刑法第271條 — 本案為毒品案，不適用殺人罪
│    └── 扣 2 分
└── ✅ 刑事訴訟法第95條告知義務 — 正確
     └── https://law.moj.gov.tw/LawClass/LawSingle.aspx?pcode=C0010001&flno=95
```

### 給分標準

實際評分規則見 [[51_人格_助教批改_v1.2.0|51_人格_助教批改]]（唯一權威來源，含級數錨定 + 查表對照 + 憲判檢查）。

| 維度 | 說明 |
|:----:|:-----|
| A 對題性 | 有無正面回答問句、是否抓到核心爭點 |
| B 結構性 | 規範→要件→涵攝→收束順暢度 |
| C 涵攝品質 | 事實是否放進要件、避免空泛 |
| D 法源精準 ⚖️ | 概念/法條正確；涉及基本權條文時需檢查憲判引用 |
| E 攻防與程序完整性 | 反方主張＋回應；程序/比例原則自然嵌入 |
| F 表達可讀性 | 句子清楚、冗詞少、考場完成度 |

> L1/L2/L3 權重及查表對照，請參閱 [[51_人格_助教批改_v1.2.0#評分量表]]。

---

## STEP 4 — 判決佐證（MCP 查詢）

批改完成後自動查詢：

1. `search_judgments()` — 以爭點關鍵字搜尋相關判決
2. `get_judgment()` — 拉取關鍵判決全文
3. 將判決觀點納入評分總結的「實務佐證」區塊

**特別檢查**：若出題涉及 §309、§310、§23 等基本權條文，強制查詢憲法法庭判決（詳見 SKILL.md 憲判強制檢查層）

---

## STEP 5 — 反思與 QC 更新（🆕）

每次完整流程結束後，系統應自動記錄：

```
反思摘要：
- 本次弱項：＿＿＿＿（如：學生漏引 113憲判3、WRITER L3 轉承語不足 5 個）
- 是否觸發憲判檢查？是/否
- 建議更新 QC 規則：＿＿＿＿
```
