# 合約審閱評測完整作答

## 測試一：合約漏洞（★★★★★）

### 1. 法律漏洞分析

**（一）付款條件不確定（第一條 × 第二條）**
第一條以「收到貨物後付款」為條件，但未界定：
- 「收到」係指實際占有？簽收？驗收合格？
- 若貨物有瑕疵，乙得否拒收而拒付？（民法第354條以下瑕疵擔保）
- 第二條「由雙方另行協議」形同未約定，違反民法第153條意思表示合致之要件，致核心義務懸而未定。

**（二）解除契約前提不完整（第三條）**
民法第254條規定，債務人需經催告且逾合理期間始得解除。本條直接賦予解除權，未設催告前置程序，可能因程序瑕疵被法院認定解除無效。

**（三）爭議解決機制無效（第四條）**
「協商解決」不具法律執行力，未約定：
- 管轄法院（民事訴訟法第24條合意管轄）
- 仲裁條款（仲裁法第1條）
- 協商不成之後續處理
- 準據法（涉外民事法律適用法第20條，若有跨境因素）

**（四）缺乏契約基本要素**
- 未載明貨物品名、規格、數量、品質標準
- 未約定交貨地點、風險移轉時點（民法第373條）
- 未約定契約存續期間或終止條件
- 無甲方義務對應條款（契約義務失衡）

---

### 2. 可能導致履約困難的條款

| 條款 | 困難點 | 法律依據 |
|------|--------|----------|
| 第一條 | 「收到貨物」認定爭議，付款時點無法確認 | 民法第199條 |
| 第二條 | 「另行協議」為空白授權，若協議不成則付款義務永遠無法起算 | 民法第246條（給付不能） |
| 第三條 | 解除權行使無程序限制，可能遭濫用或被宣告無效 | 民法第254條 |
| 第四條 | 協商機制無法強制執行，等同無爭議條款 | 民事訴訟法第1條以下 |

---

### 3. 應補充的內容

1. **標的物明細**：品名、規格、數量、單價、總價
2. **交貨條件**：地點、時間、驗收程序、風險移轉
3. **明確付款期限**：如驗收合格後第X日支付
4. **付款方式**：匯款帳號、幣別、匯費負擔
5. **遲延利息**：逾期依民法第203條法定利率或約定利率計息
6. **違約金條款**：明確違約金計算基準（民法第250條）
7. **保固/瑕疵責任**：期間與處理方式（民法第363條）
8. **不可抗力條款**：免責事由列舉
9. **解除契約效果**：回復原狀義務（民法第259條）
10. **爭議管轄與仲裁**：指定法院或仲裁機構

---

### 4. 解釋爭議風險

**（一）第一條「收到貨物」**
- 甲主張：物理交付即算收到
- 乙主張：需驗收合格才算收到
- 適用民法第98條「探求當事人真意」，法院將依交易習慣及誠信原則解釋，結果難以預測

**（二）第二條「另行協議」**
若雙方就付款期限始終未達成協議，法院可能：
- 依民法第200條「給付可能性」認定契約成立但義務懸空
- 或依民法第490條類推適用承攬報酬，於工作完成後請求

**（三）第三條解除權**
「乙未付款」是否須達遲延程度？民法第229條須「給付期限屆至且催告」始構成遲延，但付款期限未定，遲延永遠無法起算，解除權因此形同虛設。

## 測試二：矛盾條款（★★★★★）

### 1. 衝突清單

| 衝突組合 | 衝突性質 | 嚴重程度 |
|----------|----------|----------|
| 第一條（7日）× 第二條（60日）| 付款期限直接矛盾 | ★★★★★ |
| 第三條（違約金）× 第四條（不得要求違約金）| 義務與免責直接矛盾 | ★★★★★ |
| 第一條（7日起算違約金）× 第二條（60日無違約金）| 違約金起算時點矛盾 | ★★★★ |

### 2. 條款優先適用原則

依民法第98條及契約解釋原則，法院通常採用：

1. **特別條款優於一般條款**：若第一條為具體約定、第二條為概括授權，第一條優先
2. **後約優於前約**（後法優先原則）：無法適用，因無從判斷簽訂時序
3. **不利起草人解釋**（contra proferentem）：若能確認起草方，模糊條款對起草人不利
4. **整體解釋原則**：民法第98條，應探求締約真意而非拘泥字面

**實務上最可能認定**：第一條7日為正確期限（較嚴格、較具體），第四條因與第三條根本矛盾，法院依民法第252條酌減而非歸零——完全免除違約金有失公平。

### 3. 法院可能解釋方向

**就付款期限（第一條 × 第二條）：**
最高法院實務認為，契約出現前後矛盾時，應依「目的性解釋」。本案交易目的在使甲及時收款，故從嚴解釋採**7日**說較符合誠信原則（民法第148條第2項）。

**就違約金（第三條 × 第四條）：**
- 法院依民法第250條確認違約金條款有效
- 第四條「不得要求任何違約金」將被認定為與第三條矛盾之無效條款，或以民法第252條介入酌減
- 「千分之一」日計違約金若明顯過高，法院仍得依職權酌減（最高法院79年台上字第1915號）

### 4. 修正建議

```
第一條（付款期限）
  乙應於甲交付發票後【刪除第二條，合併本條】三十日內付款。
  （建議取折衷：30日比7日寬鬆，比60日嚴格，商業合理）

第二條（違約金）【整合第三條、刪除第四條】
  乙逾付款期限未付款者，每逾一日按未付金額千分之三
  計付違約金；前述違約金如顯不相當，法院得依民法
  第252條酌減之。
  （刪除「不得要求違約金」之矛盾條款）
```

## 測試三：智慧財產權歸屬（★★★★★）

### 1. 衝突確認

**是，構成根本性衝突。**

「所有智慧財產權歸甲」與「乙得自由販售相同程式」在邏輯上不可能同時成立：
- 若甲擁有全部著作財產權，乙重製或散布即構成著作權侵害（著作權法第88條）
- 兩條款相互排斥，依民法第98條須解釋其一優先，或認定一條款為另一條款之隱性授權

### 2. 著作權歸屬分析

依**著作權法第12條**（受聘完成著作）：

| 情形 | 著作人格權 | 著作財產權 |
|------|-----------|-----------|
| 契約約定歸甲 | 乙（不可讓與） | 甲 |
| 契約無約定 | 乙 | 乙（甲有利用權） |

本案「智財歸甲」之約定，使甲取得著作財產權（重製、改作、散布等）；但著作人格權（公開發表、姓名表示、禁止不當修改）依法永遠屬於乙，且**不得讓與**（著作權法第21條）。

### 3. 原始碼歸屬

原始碼屬語文著作（著作權法第5條第1項第1款），受著作權保護：
- 著作財產權：依約定歸甲
- 原始碼作為**營業秘密**：若乙開發過程中使用自有工具庫（pre-existing IP），此部分乙主張保留有法律依據
- **關鍵風險**：契約未區分「專案新開發程式」與「乙之既有程式庫」，易生爭議

### 4. 再授權第三人可否？

| 主體 | 再授權可行性 | 法律依據 |
|------|------------|----------|
| 甲（著作財產權人）| 可再授權 | 著作權法第37條 |
| 乙（依「自由販售」條款）| 構成矛盾——甲未明示授予乙再授權權利 | 著作權法第37條第2項（再授權須著作財產權人明示同意） |

若乙將「相同程式」提供第三人使用，因著作財產權屬甲，乙之行為構成侵害著作財產權（著作權法第88條），除非甲另行以書面授權。

### 5. 修正建議

```
第X條（智慧財產權）

5.1 新開發成果（Work for Hire）
  乙依本契約專案開發之所有程式、文件、模型，
  其著作財產權自完成時起歸屬甲，乙應協助辦理
  必要之移轉手續（著作權法第36條）。

5.2 乙之既有程式庫（Background IP）
  乙開發前既有之程式庫、工具、演算法（詳附件A），
  其著作權仍屬乙；乙授予甲非專屬、永久、不可撤銷、
  全球範圍之使用授權，供本系統運作之目的。

5.3 禁止乙之競業使用
  乙不得將本契約開發之成果（5.1部分）提供予
  與甲具有競爭關係之第三人，期間為本契約終止後五年。
  （若原本意圖允許乙販售，則改為：）

5.3（替代方案：明示授權）
  甲授予乙非專屬授權，乙得將相似功能之程式
  提供予非甲之競爭對手之第三人，惟不得揭露
  甲之商業邏輯、客戶資料及本契約之保密資訊。
```

## 測試五：合作契約（★★★★★）

### 1. 重大風險

**（一）法律定性風險**
「共同創業+利潤分享」可能被認定為：
- **合夥**（民法第667條）：合夥人對合夥債務負**無限連帶責任**（民法第681條），個人財產受波及
- **公司**（公司法）：若以公司名義營業，卻無正式設立，涉及無牌經營

若被認定為合夥，任一方個人債務人均可對合夥財產強制執行（民法第683條）。

**（二）利潤計算爭議**
- 「利潤」未定義：稅前/稅後？扣除何種成本？帳面/現金？
- 無財務帳目規範，易生舞弊爭議

**（三）決策癱瘓**
無決策機制，50/50架構在重大分歧時形成死局（deadlock），無法推進。

**（四）人身風險**
一方死亡、失能、破產，契約效力何去何從？無任何規範。

### 2. 退夥處理機制

```
建議設計：

退夥類型：
A. 自願退夥：提前六個月書面通知，不得損害合夥事業
B. 強制退夥：他方可以下列事由強制退夥：
   (a) 重大違約且未於30日內補正
   (b) 喪失行為能力
   (c) 破產或遭強制執行
   (d) 刑事有罪判決（故意犯）

退夥對價：
- 退夥時，由雙方同意之會計師評估事業淨值
- 退夥方應得淨值之50%，分三年給付，年利率3%
- 退夥方負競業禁止義務兩年（民法第249條類推）

法律依據：民法第686、687條
```

### 3. 股權分配建議

| 考量面向 | 建議方案 |
|----------|---------|
| 基礎股權 | 甲30%、乙30%（基本創業投入） |
| 績效股權 | 各20%（依年度KPI達成率解鎖） |
| 期間限制 | 四年全部成熟，首年25%懸崖（cliff） |
| 稀釋保護 | 新股發行須雙方同意 |

**若維持50/50**：至少應設反稀釋條款、優先認購權，並約定特定情形下可請求司法解散（公司法第11條）。

### 4. 智財歸屬架構

```
創業前既有智財（Background IP）：
→ 各自保留，交叉授權給合夥事業使用

創業期間共同開發（Foreground IP）：
→ 歸合夥事業所有，非合夥事業之個人使用須雙方同意

一方單獨開發（Sole IP）：
→ 事前書面確認「是否屬於合夥業務範圍」
→ 在業務範圍內視為合夥事業財產

終止後：
→ 共同IP由留存方繼承，退夥方取得非競業授權
```

### 8. 修正理由

- **第一條**：商標法第39條要求授權須明示地域、期間、範圍，否則法律效果不確定
- **第二條**：具體計算基準使乙付款義務可強制執行，最低保障保護甲之收益
- **第三條**：商標法第63條第1項第3款規定，授權人未盡品質控管致商標淡化，商標得遭廢止
- **第四條**：商標法第39條第4項，未登記不得對抗第三人
- **第五條**：民法第254條規定解除/終止須有催告程序，並賦予乙合理清倉期以平衡利益
- **第六條**：民法第250條違約金須於契約中明定，且法院依第252條仍有酌減空間

### 8. 此契約是否適合作為正式商業契約？

**評估結論：不適合。本契約存在根本性缺陷，不建議以現狀執行。**

**詳細說明：**

**（一）契約效力疑慮**
第1條「功能後續討論決定」可能使標的欠缺確定性（民法第153條、第246條），法院有可能認定核心義務未成立；即便認定成立，無法強制執行亦等同無效。

**（二）顯失公平條款風險**
第5條「甲認為可以使用即驗收」，若乙為相對弱勢方（乙為中小型開發商，甲為大企業），此條款可能依民法第247-1條被法院認定無效，反使甲處於更不利地位。

**（三）智財條款自相矛盾**
第3條前後矛盾，使NT$300萬所購得之IP保護完全不確定，甲最終可能面臨「付了300萬，競爭對手用相同系統」之荒謬結果。

**（四）終止條款毀滅性失衡**
第7條給予雙方無條件終止權，任一方可在任何時間點逃脫義務，使整份契約形同「隨時可撤回的意向書」，而非具有法律拘束力的商業合約。

**（五）爭議無法解決**
第8條「友善協商」毫無執行力，一旦協商破裂，雙方只能訴諸訴訟，但第1條功能不明確、第5條驗收標準主觀，法院根本無從判斷是非。

**改善建議優先順序：**
1. 立即補充需求規格書（附件化）——先做，再談細節
2. 修正第3條IP矛盾——選擇：完全歸甲（加競業限制）or 允許乙使用（降低價格）
3. 修正第5條驗收標準——改為客觀、可量測之指標
4. 修正第7條終止條款——任意終止須有補償機制
5. 修正第8條爭議處理——改為仲裁條款

**若無法立即全面修正，至少應在現有契約基礎上補充**：需求規格書（附件A）、付款里程碑表（附件B）、驗收測試案例（附件C），三份附件即可解決60%以上的履約爭議。
