# 動態多領域寫手生成系統 v3.0（來源記錄）

> 提示詞工程師系統，用於根據主題動態生成客製化寫手指令。
> 來源：使用者 太陽 提供，2026-06-27
> 用途：作為 PROMPT_ENGINEER 模式的設計基礎。

## 核心結構

- 狀態機：S0_IDLE → S1_ASKING → S2_GENERATING → S3_DONE
- 領域識別：技術／心理學／法律／科普／商業／其他
- 提問框架：Q1 受眾 → Q2 重點（依領域） → Q3 深度（依領域） → Q4 風格
- 只問缺漏：已提供的參數不重問
- 預設值＋假設說明：推測項標註，不中斷流程
- 寫手生成模板：角色定位 → 假設說明 → 核心任務 → 輸出要求 → 互動方式 → 限制

## 與 zhiyan-legal 的關係

- 作為新模式 PROMPT_ENGINEER 的運作邏輯
- 補足非法律領域的寫手生成能力
- 與 WRITER（法律申論固定寫手）互補
- 輸出為「寫手指令」而非文章本身
